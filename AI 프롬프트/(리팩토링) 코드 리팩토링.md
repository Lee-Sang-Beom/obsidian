
## 🎯 디렉터리 구조 원칙

- [x] **_views/** - 페이지 레벨 뷰
  - 완전한 화면 또는 큰 페이지 단위
  - Section들을 조합하여 완성된 화면 구성
  - 페이지 레벨의 상태 관리 및 데이터 fetching
  - 라우팅과 직접 연결

- [x] **_sections/** - 기능 단위 섹션
  - 여러 컴포넌트를 조합한 중간 크기의 블록
  - 특정 도메인 로직 포함
  - 하나의 기능 단위 담당
  - 페이지 내에서 재사용 가능

- [x] **_components/** - 순수 UI 컴포넌트
  - 재사용 가능한 작은 UI 조각
  - 비즈니스 로직이 거의 없음
  - props를 통해 데이터를 받아 표시
  - **도메인에 독립적** (UI 타입으로만 분류)
  - 다른 페이지에서도 사용 가능
  - 해당 디렉터리 하위에는 도메인 관련 용어가 포함되면 안됨 (ex: `components/approve` 불가)
