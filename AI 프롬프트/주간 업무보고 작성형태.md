- 일반 메모장에 작성하듯 기재
- 마크다운형태로 작성하지 않음. 

##### 업무내용
- 주간 업무보고 요약이라는 소제목이 위에 있음
- 예시는 아래와 같음
```null
<주간 업무보고 요약>
- ICMS 제작 프로젝트 페이지 개발 진행
- ICMS 통계 프로젝트 작성된 README.md 내용수정
- 보일러 플레이트 코드 업데이트
```

##### 업무결과
- [프로젝트명] 하위에 기재되는 업무 내용이 있음
	- 프로젝트 별로 상태, 현재 가장 큰 막힘, 지난 주 실제로 한 것을 기록하면 됨
	- 별다른 추가 요청사항이 없으면 상태는 원활, 현재 가장 큰 막힘은 없음으로 고정하여 작성

- 별도 요일 구분은 필요없음. 한 주에 뭘 했는지 총체적으로 아래처럼만 작성하는 게 목표

- 예시는 아래와 같음
```null
[보일러 플레이트]
- 상태: 원활
- 현재 가장 큰 막힘: 없음
- 지난주 실제로 한 것:
 > vite 메이저 버전 업데이트로 인한 패키지 업데이트 진행 (npx npm-check-updates -u)
 > BreadCrumb 리팩토링/기능개선

[ICMS/제작]
- 상태: 원활
- 현재 가장 큰 막힘: 없음
- 지난주 실제로 한 것:
 > 영상 제작 관리/가공 영상 목록
  - 리스트/상세 페이지 구현 
  - 테스트케이스 작성

 > 가공 콘텐츠 관리/인원 별 업로드 현황 
  - 테스트케이스 오류 수정: 월 데이터 변경으로 인한 조회 데이터 탐색오류 수정
  - 인원 별 업로드현황 페이지 캘린더 기능 수정

 > 가공 콘텐츠 관리/업로드 이미지 개수
  - 총 업로드 완료 이미지 수량 표시 영역 추가
  - 항목별 업로드 이미지 수량 영역 추가 (카테고리 분류별 업종/이미지수 계산 로직 별도 분리)
  - 초기화 버튼 클릭 시, 같은 payload여도 fetch UI가 확인되도록 변경
  - 페이지 전체 UI 스타일 재구성

 > 영상 제작 관리/영상 제작 
  - 테스트케이스 오류 수정: UI 스타일 수정으로 인한 텍스트 찾기 오류 수정

 > 영상 제작 관리/렌더링 목록
  - 테스트케이스 작성
  - 필요 기능 추가: 렌더링 과정 상태 내 제작 요청 중인 요소에 대해 리스트 체크박스 선택 기능 추가

 > 영상 제작 관리 하위 5개 페이지 연계 동작 테스트
 > 프로젝트 README.md 작성
```
