
#### 🔧 API / 기술 구조

**[1] 네이버 커머스 API 멀티셀러 구조**

지금 설계처럼 수십~수백 명 셀러를 중앙 서버 하나로 관리하려면, 네이버 커머스 API가 스토어 애플리케이션당 IP를 최대 3개까지만 허용하는 문제가 있어요. 

셀러 100명이면 100개의 토큰을 각각 관리해야 하는데, 현재 아키텍처에는 이 구조가 빠져 있어서 **네이버 공식 솔루션 파트너 등록 없이는 현재 방식대로 구현이 어렵습니다.** 

MVP를 네이버 단일 채널로 좁히더라도 이 파트너 등록 절차는 선결 과제인데, 진행하실 계획이 있으신가요?

**[2] AI CS 에이전트 - 네이버 톡톡 데이터**

AI CS 에이전트가 고객 문의에 자동 응답하려면 네이버 톡톡 데이터를 읽어야 하는데, 공식 커머스 API로는 지원되지 않아요. 

비공식 크롤링을 쓰면 정통망법 제48조 무단 접근 문제가 생겨서 **현재 계획하신 AI CS 자동 응답 기능은 공식 경로로는 구현이 불가능한 상태입니다.** 

MVP에서 이 기능을 어떻게 처리하실 예정인지 확인이 필요합니다.

**[3] KAMIS 실시간 시세 연동** 

사업계획서에서 "실시간 시세 연동"을 핵심 기능으로 제시하고 계신데, KAMIS OpenAPI는 실시간이 아니라 전날 또는 당일 오전 기준 데이터만 제공해요. 

가락시장·노량진 실시간 시세는 별도 제휴 API가 필요하고, 웹 크롤링은 각 시장 홈페이지 이용약관 위반 가능성이 있어서 **"실시간 시세"라는 표현을 서비스 기획에서 재검토하거나, 제휴 협약을 먼저 체결하셔야 할 것 같습니다.**

#### 🖼️ AI 이미지 / 콘텐츠

**[5] AI 생성 이미지 표시 의무** 

2026년 1월 22일부터 AI 기본법 제31조가 시행되면서, AI로 생성한 이미지나 텍스트를 상품 페이지에 쓸 경우 "AI 생성물"이라는 표시가 의무예요. 

특히 농수산물은 색감·신선도가 구매에 직결되기 때문에, **AI 생성 이미지와 실물 사이의 괴리로 소비자 오인이 생기면 표시광고법 위반으로도 이어질 수 있습니다.** 

실물 사진 병행 게재 또는 AI 생성 표시 방식을 어떻게 처리하실 계획인지 확인이 필요합니다.

**[6] AI 생성 콘텐츠 저작권** 

한국저작권위원회가 2025년 6월, AI가 생성한 콘텐츠는 저작권 보호 대상이 아니라는 원칙을 명확히 했어요.

즉, 플랫폼에서 만든 상세페이지 이미지나 텍스트를 **경쟁사가 그대로 복사해도 법적으로 막을 방법이 없습니다.**

상세페이지 AI 생성이 킬러 기능인 만큼, 콘텐츠 자산 보호 전략을 어떻게 가져가실지 논의가 필요합니다.

**[9] 운영팀 직접 셀러 운영 시 이해충돌** 

개선 권고안에서 제안한 것처럼 Phase 1에서 운영팀이 직접 셀러 계정 5~10개를 운영하며 쿠팡·네이버에 상품을 등록하는 구조는 콜드 스타트 문제를 푸는 좋은 방법이에요.

다만 **플랫폼 운영팀이 자사 플랫폼에 입점해 직접 판매할 경우, 공정거래법·온라인 플랫폼 공정화법상 이해충돌 고지 의무가 생길 수 있습니다.** 

이 부분은 법무 검토를 먼저 진행하신 후 Phase 1 설계에 반영하시는 게 좋을 것 같습니다.
