
## 1. 판매자(Seller) 화면

---
### SEL-01 | 온보딩 / 계정 연동 🔴 MVP

**화면 목적**: 판매자 최초 가입 및 오픈마켓 채널 연동
**원 명세**: 사업자 진위 확인 API 연동, 판매자 프로필 생성

#### 입력 정보

| 필드                | 타입       | 필수     | 비고                         |
| ----------------- | -------- | ------ | -------------------------- |
| 사업자등록번호           | text     | ✅      | 10자리, 하이픈 자동 포맷            |
| 대표자명              | text     | ✅      |                            |
| 상호명               | text     | ✅      |                            |
| 판매 채널 선택          | checkbox | ✅      | 쿠팡 / 네이버 스마트스토어            |
| 기본 프로필 (연락처, 이메일) | text     | ✅      |                            |
| 통신판매업 신고번호        | text     | ⚠️ 조건부 | 미결 질의 #10 확정 후 필수 여부 결정    |
| 식품판매업 허가번호        | text     | ⚠️ 조건부 | 식품 취급 시. 미결 질의 #10 확정 후 결정 |

#### 비즈니스 규칙

- 사업자등록번호 입력 후 **국세청 사업자 진위확인 API** 실시간 호출 → 결과 표시 (유효 / 휴폐업 / 미등록)
- 사업자 상태가 "활성"이어야 다음 단계 진행 가능
- 채널 연동: 네이버는 **OAuth 2.0** 방식, 쿠팡은 **Wing HMAC 키 입력** 방식 (API 명세서 기준)
- 연동 완료된 채널은 완료 뱃지로 표시

#### 출력 / 결과

- 가입 완료 화면
- 초기 프로필 생성 완료 메시지

#### UI 주요 요소

- 단계별 스텝 (판매자 가입 및 기본 정보 등록)

### SEL-04 | 공급가·시세 확인 🔴 MVP

**화면 목적**: 비딩 결과와 시세 비교 결과 확인, 예상 마진 확인 및 판매가 최종 조정
**원 명세**: Python이 계산한 공급가·수수료·마진을 설명 형태로 제공 / 최종 가격은 사람 검토 가능

#### 입력 정보

| 필드                | 타입     | 필수  | 비고                        |
| ----------------- | ------ | --- | ------------------------- |
| 공급가, 시세, 예상 마진 입력 | number | 선택  | Python 추천가 기본값, 판매자 수정 가능 |

> **참고**: 이 화면은 Python이 계산한 결과를 **조회·확인**하는 화면입니다. 대부분의 데이터는 출력값이며 판매자 입력은 판매가 조정만 해당합니다.

#### 비즈니스 규칙

- 마진 계산은 **Python 결정 코드**에서 수행. UI는 결과 표시만 담당
- 이상치 검증(KAMIS ±10%)은 **중개자(INT-05)** 단계에서 완료된 결과를 표시하는 것. SEL-04에서 별도 차단 로직 없음
- 최종 가격 확정은 판매자 직접 승인 버튼 클릭 필요
- 판매가가 마진 0 이하로 설정될 경우 저장 불가 + 경고 표시

#### 출력 / 결과

| 항목           | 표시 내용                                  |
| ------------ | -------------------------------------- |
| 공급사별 응답가 비교표 | 공급사명 / 응답가 / KAMIS 시세 대비 차이(%) / 분배 비율 |
| 마진 계산 내역     | 항목별 비용 구성 + 예상 마진액·마진율                 |
| 판매가 입력 필드    | Python 추천가 기본값 표시                      |
| 채널별 예상 수수료   | 쿠팡 / 네이버 각각 정보                         |

#### UI 주요 요소

- 비딩 결과 및 시세 비교결과 표시 UI
- 마진 계산 상세 아코디언 (기본 접힘: 펼치면 계산식 확인)
- 필요 시 판매가 직접 입력 필드 구성(실시간 마진 재계산)

### SEL-05 | 상세페이지 초안 검토·수정 🔴 MVP

**화면 목적**: AI가 생성한 상세페이지 초안을 판매자가 검토하고 수정 요청하는 화면
**원 명세**: AI가 초안 생성, 판매자가 수정 요청 가능 / 판매자 검토 필수

#### 입력 정보

| 필드          | 타입       | 필수  | 비고                |
| ----------- | -------- | --- | ----------------- |
| 수정 요청 채팅 입력 | textarea | 선택  | 자연어로 수정 요청        |
| 이미지 업로드     | file     | 선택  | 상품 실사 이미지, 최대 10장 |
| 옵션 구성 수정    | text     | 선택  |                   |

#### 비즈니스 규칙

- AI가 상세페이지 초안 생성 (타이틀, 상세설명, 검색 키워드, 옵션 구성 포함)
- 판매자가 자연어로 수정 요청 → AI가 재생성
- **판매자 검토 및 최종 승인 버튼 클릭이 필수** — 미승인 시 등록 API 호출 불가
- 이미지는 스토리지에 업로드, URL만 DB에 저장
- ⚠️ **법적 고지 필수 (미결 질의 #10)**: "AI 초안은 참고용, 최종 내용의 정확성은 판매자 책임" 문구 화면에 표시

#### 출력 / 결과

| 항목          | 내용                          |
| ----------- | --------------------------- |
| AI 생성 초안    | 타이틀 / 상세설명 / 검색 키워드 / 옵션 구성 |
| 수정 반영 상세페이지 | 채팅 수정 요청 반영 후 재렌더링          |
| 검토 확인 버튼    | 승인 버튼 클릭 후 "등록 요청 가능" 상태 전환 |

#### UI 주요 요소

- 좌: 상세페이지 미리보기 / 우: 수정 요청 채팅 패널 (분할 레이아웃)
- 채널별 미리보기 탭 (쿠팡 / 네이버)
- 이미지 드래그앤드롭 업로드 + 순서 변경
- "AI에게 수정 요청" 채팅 입력창
- "이 버전으로 확정" CTA 버튼
- AI 초안 법적 고지 문구 (하단 고정)

## 2. 공급자(Supplier) 화면

### SUP-01·02 | 공급자 프로필·품목 등록 🔴 MVP

**화면 목적**: 공급자 최초 등록 및 취급 품목·기준 가격 입력

|화면 ID|원 명세|
|---|---|
|SUP-01|업체 정보, 카테고리, 납품 조건 / 공급사 풀 관리와 연동|
|SUP-02|상품명, 규격, 가격, MOQ / 기준 상품 정보 관리|

#### 입력 정보

| 필드         | 타입       | 필수  | 비고                                       |
| ---------- | -------- | --- | ---------------------------------------- |
| 업체명        | text     | ✅   |                                          |
| 사업자등록번호    | text     | ✅   | 국세청 API 진위 확인                            |
| 주요 취급 카테고리 | checkbox | ✅   | 쌀·잡곡 / 냉동수산 / 건기식 / 수입과일 등               |
| 납품 가능 지역   | checkbox | ✅   |                                          |
| 담당자 연락처    | text     | ✅   |                                          |
| 품목별 기준가 입력 | 테이블      | ✅   | 상품명 / 규격 / 기준가 / MOQ                     |

#### 비즈니스 규칙

- 프로필 등록 후 **운영자 승인 대기** 상태 → 승인 전까지 비딩 요청 미발송
- 품목 기준가는 KAMIS 시세와 비교 표시 (경고만, 강제 차단 아님)

#### 출력 / 결과

- 공급자 프로필 등록 완료 화면
- 운영자 승인 대기 상태 표시
- 등록된 품목 리스트 + 저장 완료 확인

#### UI 주요 요소

- 최종 응답 발송 완료 확인

- 3단 레이아웃: 좌(고객 문의 원문) / 중(AI 분류 + 유사 케이스) / 우(최종 응답 작성)

- 발송 전 "최종 확인" 모달
