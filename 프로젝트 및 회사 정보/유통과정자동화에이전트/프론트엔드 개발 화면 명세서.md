
## 1. 판매자(Seller) 화면


### SEL-01 | 온보딩 / 계정 연동 🔴 MVP

**화면 목적**: 판매자 최초 가입 및 오픈마켓 채널 연동 **원 명세**: 사업자 진위 확인 API 연동, 판매자 프로필 생성

#### 입력 정보

|필드|타입|필수|비고|
|---|---|---|---|
|사업자등록번호|text|✅|10자리, 하이픈 자동 포맷|
|대표자명|text|✅||
|상호명|text|✅||
|판매 채널 선택|checkbox|✅|쿠팡 / 네이버 스마트스토어 / 11번가|
|기본 프로필 (연락처, 이메일)|text|✅||
|통신판매업 신고번호|text|⚠️ 조건부|⚠️ 미결 질의 #10 확정 후 필수 여부 결정|
|식품판매업 허가번호|text|⚠️ 조건부|⚠️ 식품 취급 시. 미결 질의 #10 확정 후 결정|

#### 비즈니스 규칙

- 사업자등록번호 입력 후 **국세청 사업자 진위확인 API** 실시간 호출 → 결과 표시 (유효 / 휴폐업 / 미등록)
- 사업자 상태가 "활성"이어야 다음 단계 진행 가능
- 채널 연동: 네이버는 **OAuth 2.0** 방식, 쿠팡은 **Wing HMAC 키 입력** 방식 (API 명세서 기준)
- 연동 완료된 채널은 완료 뱃지로 표시

#### 출력 / 결과

- 가입 완료 화면 + 텔레그램 봇 연결 안내
- 연동된 채널 목록 및 상태 (연동 완료 / 실패 / 미연동)
- 초기 프로필 생성 완료 메시지

#### UI 주요 요소

- 단계별 스텝퍼 (사업자 확인 → 채널 연동 → 텔레그램 연결)
- 국세청 API 로딩 스피너
- OAuth 팝업 윈도우 (네이버), HMAC 키 입력 마스킹 필드 (쿠팡)

### SEL-04 | 공급가·시세 확인 🔴 MVP

**화면 목적**: 비딩 결과와 시세 비교 결과 확인, 예상 마진 확인 및 판매가 최종 조정 **원 명세**: Python이 계산한 공급가·수수료·마진을 설명 형태로 제공 / 최종 가격은 사람 검토 가능

#### 입력 정보

|필드|타입|필수|비고|
|---|---|---|---|
|판매가 조정 입력|number|선택|Python 추천가 기본값, 판매자 수정 가능|
|채널별 판매가 개별 설정 여부|toggle|선택|채널마다 다른 가격 설정 시|

> **참고**: 이 화면은 Python이 계산한 결과를 **조회·확인**하는 화면입니다. 대부분의 데이터는 출력값이며 판매자 입력은 판매가 조정만 해당합니다.

#### 비즈니스 규칙

- 마진 계산은 **Python 결정 코드**에서 수행. UI는 결과 표시만 담당
    - 계산 항목: 매입가 + 택배비 + 포장비 + 플랫폼 수수료 + 오픈마켓 수수료 = 원가 합계 / 예상 마진
    - ⚠️ **구체적 수치(택배비, 수수료율 등)는 운영 정책 확정 필요** — 현재 미결 질의 #6, 8 연관
- 이상치 검증(KAMIS ±10%)은 **중개자(INT-05)** 단계에서 완료된 결과를 표시하는 것. SEL-04에서 별도 차단 로직 없음
- 최종 가격 확정은 판매자 직접 승인 버튼 클릭 필요
- 판매가가 마진 0 이하로 설정될 경우 저장 불가 + 경고 표시

#### 출력 / 결과

|항목|표시 내용|
|---|---|
|공급사별 응답가 비교표|공급사명 / 응답가 / KAMIS 시세 대비 차이(%) / 분배 비율|
|KAMIS 시세 기준선|최근 7일 평균, 조회일 표시|
|마진 계산 내역|항목별 비용 구성 + 예상 마진액·마진율|
|판매가 입력 필드|Python 추천가 기본값 표시|
|채널별 예상 수수료|쿠팡 / 네이버 / 11번가 각각|

#### UI 주요 요소

- 공급사 응답가 비교 바 차트 (KAMIS 기준선 표시)
- 마진 계산 상세 아코디언 (기본 접힘)
- 판매가 직접 입력 필드 (실시간 마진 재계산)
- "가격 확정" 버튼 (CTA)

### SEL-05 | 상세페이지 초안 검토·수정 🔴 MVP

**화면 목적**: AI가 생성한 상세페이지 초안을 판매자가 검토하고 수정 요청하는 화면 **원 명세**: AI가 초안 생성, 판매자가 수정 요청 가능 / 판매자 검토 필수

#### 입력 정보

|필드|타입|필수|비고|
|---|---|---|---|
|수정 요청 채팅 입력|textarea|선택|자연어로 수정 요청|
|이미지 업로드|file|선택|상품 실사 이미지, 최대 10장|
|옵션 구성 수정|테이블 편집|선택|용량/중량 옵션 등|

#### 비즈니스 규칙

- AI가 상세페이지 초안 생성 (타이틀, 상세설명, 검색 키워드, 옵션 구성 포함)
- 판매자가 자연어로 수정 요청 → AI가 재생성
- **판매자 검토 및 최종 승인 버튼 클릭이 필수** — 미승인 시 등록 API 호출 불가
- 이미지는 스토리지에 업로드, URL만 DB에 저장
- ⚠️ **법적 고지 필수 (미결 질의 #10)**: "AI 초안은 참고용, 최종 내용의 정확성은 판매자 책임" 문구 화면에 표시

#### 출력 / 결과

|항목|내용|
|---|---|
|AI 생성 초안|타이틀 / 상세설명 / 검색 키워드 / 옵션 구성|
|수정 반영 상세페이지|채팅 수정 요청 반영 후 재렌더링|
|채널별 미리보기|쿠팡 / 네이버 / 11번가 레이아웃 탭 전환|
|최종 승인 완료 상태|승인 버튼 클릭 후 "등록 요청 가능" 상태 전환|

#### UI 주요 요소

- 좌: 상세페이지 미리보기 / 우: 수정 요청 채팅 패널 (분할 레이아웃)
- 채널별 미리보기 탭 (쿠팡 / 네이버 / 11번가)
- 이미지 드래그앤드롭 업로드 + 순서 변경
- "AI에게 수정 요청" 채팅 입력창
- "이 버전으로 확정" CTA 버튼
- AI 초안 법적 고지 문구 (하단 고정)

### SEL-06 | 멀티채널 상품 등록 요청 + 결과 조회 🔴 MVP

**화면 목적**: 승인된 상세페이지의 오픈마켓 등록 요청 및 결과 확인 **원 명세**: API 호출 전 첫 판매 여부·정책 여부 확인 / 신규 셀러 첫 등록 승인 필요

> SEL-06(등록 요청)과 SEL-07(등록 결과 조회)는 동일 화면에서 처리

#### 입력 정보

|필드|타입|필수|비고|
|---|---|---|---|
|채널 선택|checkbox|✅|쿠팡 / 네이버 / 11번가 (복수 선택)|
|재고 정책|radio|✅|무한 재고 / 수량 직접 입력|
|판매 시작일|date|선택|기본값: 즉시|

#### 비즈니스 규칙

- **신규 셀러 첫 등록**: 운영자 승인 완료 후에만 API 호출 가능 → 승인 대기 상태 표시
- 기존 셀러: 즉시 등록 API 호출
- 쿠팡 등록 순서: 상품 생성 → 승인 요청 → 10분 단위 상태 폴링 (쿠팡 API 명세 기준)
- 네이버 등록: 원상품 등록 → 그룹 상품 전환 (해당 시, 네이버 API 명세 기준)
- 등록 실패 시: 실패 사유 표시 + 재시도 버튼

#### 출력 / 결과

|항목|내용|
|---|---|
|채널별 등록 상태|처리중 / 등록완료 / 승인대기 / 실패|
|상품 ID|채널별 고유 상품 번호|
|오픈마켓 상품 링크|등록 완료 시 바로가기 버튼|
|실패 사유|API 오류 메시지 한국어 변환 표시|

#### UI 주요 요소

- 채널별 등록 상태 카드 (아이콘 + 상태 뱃지)
- 진행 중 스피너 (쿠팡 승인 대기 최대 10분)
- "오픈마켓에서 보기" 외부 링크 버튼

### SEL-08 | 주문 현황 조회 (배송·출고 알림 통합) 🟡 2단계

**화면 목적**: 발생한 주문과 처리 현황 확인. SEL-09 배송/출고 알림 수신 기능 통합 **원 명세**: 5분 폴링 결과를 셀러에게 알림 / 주문 접수/출고/배송 상태

#### 입력 정보 (필터)

|필드|타입|비고|
|---|---|---|
|채널 필터|select|전체 / 쿠팡 / 네이버 / 11번가|
|주문 상태 필터|select|결제완료 / 출고중 / 배송중 / 배송완료 / 취소·반품|
|날짜 범위|date range|기본값: 최근 7일|
|주문번호 검색|text||

#### 비즈니스 규칙

- **5분 폴링**: 쿠팡 발주서 API (분단위, 24h 범위), 네이버 상품주문 조회 API (최대 300건) 결과를 중앙 DB에 취합 후 표시 (API 명세서 기준)
- 마지막 폴링 시각 타임스탬프 표시
- 출고 완료 및 송장 등록 시 인앱 알림 표시 (SEL-09 통합)
- 취소·반품은 별도 탭으로 분리

#### 출력 / 결과

|항목|내용|
|---|---|
|주문 리스트|주문번호 / 상품명 / 채널 / 주문수량 / 결제금액 / 상태 / 공급사 배정 여부|
|건별 상세|구매자 정보(마스킹) / 배송지 / 송장번호 / 공급사|
|통계 요약|오늘 주문 건수 / 출고 대기 / 배송중 / 완료 카운트|

#### UI 주요 요소

- 탭: 전체 / 출고대기 / 배송중 / 완료 / 취소반품
- 리스트 테이블 (페이지네이션)
- 마지막 업데이트 시각 표시 + 수동 새로고침 버튼
- 건별 상세 슬라이드 패널

### SEL-10·11·12 | 리뷰·CS 현황 + 마케팅 초안 🟡 2단계

**화면 목적**: 리뷰 요약 확인, CS 처리 현황 공유, 마케팅 콘텐츠 초안 선택 **원 명세 SEL-10**: 긍정/부정 리뷰 분류, 핵심 키워드 추출 / 리뷰 요약, 개선 포인트 **원 명세 SEL-11**: 단순/고위험 문의 분류 후 상태 공유 / 처리 현황, 후속 조치 **원 명세 SEL-12**: AI가 카피 초안 생성 / 실제 게시 행위는 판매자 직접 수행

#### 입력 정보

|필드|타입|비고|
|---|---|---|
|상품 선택|select|등록된 상품 목록|
|마케팅 초안 선택|radio|AI 생성 카피 중 선택|
|채널 선택 (마케팅)|checkbox|인스타그램 / 틱톡 / 블로그|

#### 비즈니스 규칙

- 리뷰: 긍정/부정 자동 분류 (AI), 핵심 키워드 추출
- CS 문의: 단순 문의 / 고위험 (품질·환불) 분류 → 고위험은 운영자/CS 처리 중 상태로 표시
- **마케팅 콘텐츠 실제 게시는 판매자가 직접 수행** → 초안 복사 버튼만 제공 (자동 게시 금지)

#### 출력 / 결과

|항목|내용|
|---|---|
|리뷰 요약|긍정/부정 건수, 별점 평균, 핵심 키워드|
|개선 포인트|AI가 추출한 주요 불만 사항|
|CS 처리 현황|문의 건수 / 처리 상태별 카운트 / 건별 상태|
|마케팅 초안|채널별 카피 초안 카드 + "복사" 버튼|

#### UI 주요 요소

- 리뷰 긍정/부정 도넛 차트
- CS 문의 상태 뱃지 리스트 (처리중 / 완료 / 에스컬레이션)
- 마케팅 초안 카드 비교 선택 UI + "복사" 버튼

### SEL-13·14 | 판매 성과·정산 조회 🟢 3단계

**화면 목적**: 채널별 매출·마진·환불 현황 및 정산 예정금 확인 **원 명세 SEL-13**: 거래/리뷰/클레임 데이터 집계 / 성과 요약 리포트 **원 명세 SEL-14**: 중앙 정산 구조 반영, 지급 대기 현황 표시 / 정산 예정금, 지급 내역

#### 입력 정보 (필터)

|필드|타입|비고|
|---|---|---|
|기간 선택|date range|주·월·분기·직접 입력|
|채널 필터|select|전체 / 쿠팡 / 네이버 / 11번가|
|상품 필터|select|전체 / 개별 상품|

#### 비즈니스 규칙

- 쿠팡 정산: 구매확정 또는 배송완료+7일 기준 (쿠팡 API `GET /v1/revenue-history`)
- 네이버 정산: 수수료 상세 + 부가세 건별 조회 (네이버 API `GET /v1/pay-settle/settle/commission-details`)
- 환불 건은 매출에서 차감 표시
- ⚠️ 정산 구조 (구조① vs 구조②)에 따라 화면 표시 방식이 달라짐 — **미결 질의 #7 확정 필요**
- 정산 확정은 운영 정책에 따름 (미확정 건은 "예정" 뱃지)

#### 출력 / 결과

|항목|내용|
|---|---|
|채널별 매출·마진·환불 그래프|기간별 라인 차트|

## 6. 미결 질의사항 (개발 착수 전 확정 필요)

원본 명세서 기준 미결 항목과 이번 검토에서 새로 식별된 항목을 통합합니다.

### 판매자(Ant) 관련

1. **최소 주문 수량(MOQ)**: 판매자 상품 주문 시 최소 수량 정책 → SUP-05 재고 화면 및 SEL-06 등록 화면 MOQ 필드 추가 여부
2. **반품 재고 부담**: 반품 물건 공급자 전달 구조에서 재고 부담 수수료 발생 여부 → SUP-12 정산 화면 수수료 항목

### 중개자(Ant) 관련

3. **소량 주문 배분 처리**: 6:3:1 배분 시 1개·3개 주문 처리 방식 (최저가 단독 출고 vs 라운드로빈) → INT-06·08·09 화면 직접 영향
4. **판매자 상품 구조**: 6:3:1 배분 시 3개 별도 상품 등록 vs 단일 상품에 3개 공급사 출고 → SEL-06 등록 화면, INT-08 배분 확정 화면 구조 분기

### 공급자(Ant) 관련

5. **최초 공급자 상품 등록 데이터 형식**: 오픈마켓 등록 기준 포맷 필요 여부 → SUP-01·02 품목 등록 폼 필드 설계 (카테고리 코드, 인증 번호 등)

### 공통 관련

6. **후불 거래 조건 리스크 부담 주체**: 중개자가 공급자에게 선불 지급 후 판매자에게 비용 회수하는 구조인지 → INT-14 정산 정책 화면 설계
7. **정산 흐름 구조 확정**: 구조① (판매자→중개자→공급자) vs 구조② (판매자→공급자 직거래, 중개자 수수료만) → INT-14, SEL-14, SUP-12 화면 설계 전면 분기
8. **정산 시점 불일치 처리**: 판매자 카드 결제 → 중개자 현금 → 공급자 지급 시점 차이 → INT-14 현금흐름 갭 시뮬레이션 파라미터
9. **에스크로 수수료 부담 주체**: 판매자→중개자→공급자 에스크로 연결 시 수수료 부담자

### 법적 리스크 관련 (UI 설계 직접 영향)

10. **상세페이지 생성 책임 고지**: SEL-05 화면에 "AI 초안은 참고용, 최종 내용의 정확성은 판매자 책임" 문구 표시 방식 및 위치 확정
11. **가격/마진 계산 영향력 고지**: SEL-04 화면에 "참고용 계산값, 최종 판단은 판매자 책임" 면책 문구 표시 방식 확정
12. **CS 초안 생성 책임**: INT-12 화면에서 AI 초안 영역과 최종 응답 작성 영역의 시각적 분리 방식 확정
