## 0. 공통 / 어드민 접근 제어

**경로:** `/admin/*`  
**접근 권한:** 어드민 계정 전용 (일반 사용자 완전 차단)  
**목적:** 어드민 페이지 전반의 인증·인가 처리 및 공통 UI 동작을 보장한다

---

### 기능 목록

|#|기능|설명|
|---|---|---|
|1|비로그인 접근 차단|비로그인 상태에서 /admin 접근 시 로그인 페이지로 리다이렉트한다|
|2|일반 사용자 접근 분리|일반 사용자 계정과 어드민 콘솔 접속이 완전히 분리되어야 한다|
|3|어드민 로그인|어드민 계정으로 정상 로그인 후 콘솔에 진입할 수 있다|
|4|로그아웃|로그아웃 버튼 클릭 시 세션이 완전히 만료되고 로그인 페이지로 이동한다|
|5|하위 URL 권한 검증|각 하위 메뉴(/admin/*) URL 직접 접근 시에도 권한 검증이 수행된다|
|6|사이드바 네비게이션|사이드바의 모든 메뉴 링크가 정상 동작하며 해당 페이지로 이동한다|
|7|헤더 인증 상태 표시|로그인 여부에 따라 상단 헤더에 적절한 계정 정보가 표시된다|
|8|접근성 — 색상 대비|어두운 배경의 텍스트 색상 대비(Color Contrast)가 적절하게 유지된다|
|9|알림 표시|알림 발생 시 헤더에 해당 내용이 표시되며, 모두 읽음 처리가 가능하다|
|10|로그인 중 로그인 페이지 접근|로그인 상태에서 로그인 페이지 접속 시 메인 페이지로 자동 이동한다|
|11|로고 클릭 이동|로고 클릭 시 메인 페이지 또는 로그인 페이지로 이동한다|
|12|권한 실시간 재검증|어드민 권한 박탈 시 현재 세션에서 즉시 관리자 기능 접근이 차단된다|

---

### 사용자 흐름

#### [흐름 1] 정상 로그인 및 콘솔 진입

```
/admin 접속 (비로그인) → 로그인 페이지로 리다이렉트
  └─► 어드민 계정 입력 → [로그인] 클릭
        └─► 어드민 콘솔(/admin) 진입
              └─► 사이드바 메뉴를 통해 각 기능 페이지로 이동
```

#### [흐름 2] 로그아웃

```
헤더 [로그아웃] 버튼 클릭
  └─► 세션 만료 → 로그인 페이지로 이동
        └─► 이후 /admin 직접 접근 시 로그인 페이지로 리다이렉트
```

#### [흐름 3] 권한 박탈 시 실시간 차단

```
어드민 A의 권한이 다른 관리자에 의해 박탈됨
  └─► 어드민 A의 현재 세션에서 관리자 기능(업데이트 등) 즉시 차단
        └─► 다음 페이지 이동 또는 액션 시도 시 로그인 페이지로 리다이렉트
```

## 1. 운영 대시보드

**경로:** `/admin`  
**접근 권한:** 어드민 전용  
**목적:** 플랫폼 주요 지표를 한눈에 확인하고 긴급 제어를 수행한다

---

### 기능 목록

|#|기능|설명|
|---|---|---|
|1|로딩 인디케이터|데이터 로딩 중 스켈레톤 UI 또는 로딩 인디케이터를 표시한다|
|2|주요 지표 표시|대시보드 진입 시 주요 지표(거래 수, 사용자 수, 매출 등)가 표시된다|
|3|긴급 거래 정지|긴급 거래 정지 버튼 클릭 시 확인 모달이 표시되고, 승인 시 전체 거래가 중단된다. 이벤트 불가 상태라면 별도 안내 메시지를 노출한다|
|4|에스크로 자금 동결|에스크로 보관 자금 동결 버튼 클릭 시 이벤트가 발생한다. 이벤트 불가 상태라면 별도 안내 메시지를 노출한다|
|5|데이터 로딩 실패 처리|데이터 로딩 실패 시 오류 메시지가 표시된다|
|6|거래액/트래픽 추이 차트|거래액 및 트래픽 추이 차트가 정상 렌더링되며, 실제 DB 값과 일치한다|
|7|방문자 추이 차트|방문자 추이 차트가 정상 렌더링된다|
|8|빠른 접근 링크|각 섹션으로 이동하는 바로가기(Quick Link)가 정상 동작한다|

---

### 사용자 흐름

#### [흐름 1] 긴급 거래 정지

```
대시보드에서 [긴급 거래 정지] 버튼 클릭
  └─► 확인 모달 표시 및 전체 거래 중단 처리 → 상태 변경 완료 피드백 노출
```

#### [흐름 2] 에스크로 자금 동결

```
에스크로 보관 자금 영역에서 [자금 동결] 버튼 클릭
  └─► 확인 모달 표시 → 승인 시 해당 자금 동결 처리
        └─► 동결 완료 피드백 노출
```

## 2. 거래 운영 관리

**경로:** `/admin/deals`  
**접근 권한:** 어드민 전용  
**목적:** 진행 중인 거래를 모니터링하고 단계를 제어한다

---

### 기능 목록

| #   | 기능          | 설명                                                            | 대상  |
| --- | ----------- | ------------------------------------------------------------- | --- |
| 1   | 거래 목록 조회    | 전체 거래 목록이 표시된다 (Status, Listing, Amount, Seller, Buyer, Date) | 어드민 |
| 2   | 상태별 필터      | 거래 상태(진행 중 / 완료 / 취소)별 필터가 동작한다                               | 어드민 |
| 3   | 페이지네이션      | 페이지네이션 또는 무한 스크롤이 정상 동작한다                                     | 시스템 |
| 4   | 거래 상세 조회    | 개별 거래 클릭 시 단계별 진행 현황, 채팅, 파일 등 상세 정보가 표시된다                    | 어드민 |
| 5   | 거래 단계 강제 변경 | 어드민이 거래 단계를 강제로 변경할 수 있다                                      | 어드민 |
| 6   | 거래 강제 취소/완료 | 어드민이 거래를 강제 취소 또는 완료 처리할 수 있다                                 | 어드민 |
| 7   | 채팅 내역 열람    | 거래 내 채팅 내역을 어드민이 열람할 수 있다                                     | 어드민 |
| 8   | 데이터룸 열람     | 거래 내 업로드된 파일(데이터룸)을 열람할 수 있다                                  | 어드민 |

---

### 사용자 흐름

#### [흐름 1] 거래 상세 조회 및 단계 변경

```
/admin/deals 접속
  └─► 거래 목록 확인 (상태별 필터 적용 가능)
        └─► 특정 거래 클릭 → 상세 정보 (단계, 채팅, 파일) 확인
              └─► [단계 변경] 또는 [강제 취소/완료] 선택
                    └─► 확인 모달 → 처리 완료 피드백
```

## 3. 분쟁 & 리스크

**경로:** `/admin/disputes`  
**접근 권한:** 어드민 전용  
**목적:** 접수된 분쟁을 처리하고 리스크 거래를 동결한다

---

### 기능 목록

| #   | 기능       | 설명                                                                |
| --- | -------- | ----------------------------------------------------------------- |
| 1   | 분쟁 목록 조회 | 접수된 분쟁 목록이 표시된다 (분쟁 접수일, 관련 거래, 당사자 정보 포함)                        |
| 2   | 분쟁 상세 조회 | 개별 분쟁의 상세 내용을 조회할 수 있다                                            |
| 3   | 분쟁 상태 변경 | 어드민이 분쟁 상태를 해결 / 기각 등으로 변경할 수 있다                                  |
| 4   | 처리 결과 기록 | 처리 결과 및 사유를 기록할 수 있다                                              |
| 5   | 처리 완료 알림 | 분쟁 처리 완료 시 관련 사용자에게 알림이 발송된다                                      |
| 6   | 전체 거래 동결 | 전체 거래 동결 버튼 클릭 시 이벤트가 동작한다. 분쟁에 포함된 거래만 동결되며, 포함되지 않은 거래는 정상 진행된다 |

---

### 사용자 흐름

#### [흐름 1] 분쟁 처리

```
/admin/disputes 접속
  └─► 분쟁 목록 확인
        └─► 특정 분쟁 클릭 → 상세 조회 (당사자, 거래 정보, 경위 확인)
              └─► 상태 변경(해결/기각) 선택 → 처리 사유 입력 → 저장
                    └─► 처리 완료 → 관련 사용자에게 알림 자동 발송
```

#### [흐름 2] 전체 거래 동결

```
[전체 거래 동결] 버튼 클릭 → 확인 모달 표시
  └─► 승인 → 분쟁에 포함된 거래만 동결 처리
        └─► 동결 완료 피드백 노출 (미포함 거래는 정상 진행 유지)
```

## 4. 매출/정산 관리

**경로:** `/admin/financials`  
**접근 권한:** 어드민 전용  
**목적:** 플랫폼 매출 현황과 정산 내역 확인

---

### 기능 목록

| #   | 기능                 | 설명                                          |
| --- | ------------------ | ------------------------------------------- |
| 1   | 수익 현황 표시           | 수익 콘텐츠(플랫폼 수수료 수익, 총 거래액 등) 정보가 표시된다        |

| 5   | Stripe 대시보드 열기     | Stripe 대시보드 열기 버튼 클릭 시 외부 Stripe 대시보드로 이동한다 |

### 사용자 흐름

```
/admin/financials 접속
  └─► 수익 콘텐츠 및 정산 내역 확인
        └─► [Stripe 대시보드 열기] 클릭 → 외부 Stripe 대시보드로 이동
```

## 5. 매물 품질 관리

**경로:** `/admin/listings`  
**접근 권한:** 어드민 전용  
**목적:** 등록된 매물을 검토하고 승인 또는 반려 처리한다

---

### 기능 목록

| #   | 기능             | 설명                                                           |
| --- | -------------- | ------------------------------------------------------------ |
| 1   | 로딩 인디케이터       | 데이터 로딩 중 스켈레톤 UI 또는 로딩 인디케이터를 표시한다                           |
| 2   | 매물 목록 조회       | 전체 매물 목록이 표시된다 (매물명, 등록일, 판매자, 판매가, 상태)                      |
| 3   | 상태별 필터         | 매물 상태(검토 중 / 승인 / 반려 / 판매완료)별 필터가 동작한다                       |
| 4   | 매물 상세 조회       | 개별 매물의 상세 정보를 어드민 페이지에서 확인할 수 있다                             |
| 5   | 매물 승인          | 어드민이 매물을 승인(검토 중 → 게시) 처리할 수 있다                              |
| 6   | 매물 반려          | 어드민이 매물을 반려 처리하고 사유를 입력할 수 있다. 반려 성공 시 판매자에게 사유와 함께 알림이 발송된다 |
| 7   | 거래 없는 승인 매물 반려 | 거래가 없는 승인 매물 반려 시 게시 취소 및 판매자 알림으로 처리된다                      |
| 8   | 거래 중 매물 반려 경고  | 거래 진행 중인 매물 반려 시도 시 어드민에게 경고 모달이 표시된다                        |
| 9   | 에스크로 입금 전 반려   | 에스크로 입금 전 거래 중 매물 반려 시 거래 강제 취소 후 게시 취소 처리된다                 |
| 10  | 에스크로 입금 후 반려   | 에스크로 입금 후 거래 중 매물은 즉시 반려가 불가하며, 환불 처리 흐름으로 안내된다              |

---

### 매물 반려 처리 분기

|매물 상태|처리 방식|
|---|---|
|검토 중 매물|반려 사유 입력 → 반려 처리 → 판매자 알림 발송|
|거래 없는 승인 매물|게시 취소 → 판매자 알림 발송|
|에스크로 입금 전 거래 중인 승인 매물|경고 모달 표시 → 확인 시 거래 강제 취소 → 게시 취소|
|에스크로 입금 후 거래 중인 승인 매물|즉시 반려 불가 안내 → 환불 처리 흐름으로 이동|

---

### 사용자 흐름

#### [흐름 1] 매물 검토 및 승인

```
/admin/listings 접속
  └─► 매물 목록 확인 → 검토 중 매물 필터 선택
        └─► 개별 매물 클릭 → 상세 정보 확인
              └─► [승인] 클릭 → 게시 처리 완료
```

#### [흐름 2] 매물 반려

```
개별 매물 상세에서 [반려] 클릭
  ├─► 검토 중 매물 → 반려 사유 입력 → 저장 → 판매자 알림 발송
  ├─► 거래 없는 승인 매물 → 게시 취소 처리 → 판매자 알림 발송
  ├─► 에스크로 입금 전 거래 중 매물 → 경고 모달 → 확인 시 거래 취소 + 게시 취소
  └─► 에스크로 입금 후 거래 중 매물 → 즉시 반려 불가 안내 → 환불 처리 흐름으로 이동
```

## 6. 사용자 관리

**경로:** `/admin/users`  
**접근 권한:** 어드민 전용  
**목적:** 사용자 목록을 조회하고 제재 및 권한을 관리한다

---

### 기능 목록

| #   | 기능            | 설명                                                        |
| --- | ------------- | --------------------------------------------------------- |
| 1   | 사용자 목록 조회     | 전체 사용자 목록이 표시된다 (유저정보, 등급, 상태, 가입일, 적용가능 액션)              |
| 2   | 사용자 검색        | 이메일 또는 닉네임으로 사용자를 검색할 수 있다                                |
| 3   | 사용자 정지 (Ban)  | 다.어드민이 특정 사용자의 서비스 이용을 제한할 수 있                            |
| 4   | 정지 해제         | 어드민이 사용자 정지를 해제할 수 있다                                     |
| 5   | 강제 탈퇴         | 어드민이 사용자를 강제 탈퇴 처리할 수 있다                                  |
| 6   | 어드민 권한 부여     | 일반 사용자를 어드민으로 권한 변경할 수 있다                                 |
| 7   | 어드민 권한 박탈     | 어드민 권한을 일반 사용자로 변경할 수 있다. 권한 박탈 시 해당 세션에서 즉시 관리자 기능이 차단된다 |
| 8   | 권한 변경 후 접속 검증 | 어드민으로 전환된 회원은 관리자 사이트 로그인이 가능하며, 권한 박탈 시 로그인이 불가하다        |

---

### 사용자 흐름

#### [흐름 1] 사용자 정지 처리

```
/admin/users 접속
  └─► 사용자 검색 또는 목록에서 대상 선택
        └─► [Ban] 버튼 클릭 → 확인 모달
              └─► 정지 처리 완료 → 해당 사용자 서비스 접근 제한
```

#### [흐름 2] 어드민 권한 변경

```
대상 사용자 선택 → [Make Admin] 혹은 [Remove Admin] 클릭 
  ├─► 권한 부여 시 → 해당 사용자 /admin 로그인 가능
  └─► 권한 박탈 시 → 현재 세션에서 즉시 관리자 기능 차단
                      └─► 다음 액션 시 로그인 페이지로 리다이렉트
```

## 9. 시스템 설정

**경로:** `/admin/settings`  
**접근 권한:** 어드민 전용  
**목적:** 플랫폼 수수료, Feature Flag, 점검 모드 등 시스템 설정을 관리한다

---

### 기능 목록

|#|기능|설명|
|---|---|---|

|2|설정 저장 피드백|설정 변경 저장 후 완료 피드백(Toast 등)이 표시된다|

|5|점검 모드 안내 페이지|점검 모드 활성화 시 일반 사용자에게 점검 안내 페이지가 표시된다|

---

### 사용자 흐름

#### [흐름 1] 수수료 변경

```
/admin/settings 접속
  └─► 수수료 비율(%) 변경 입력 → [Save Changes] 클릭
        └─► 완료 Toast 피드백 노출
              └─► /admin/financials 에서 변경된 수수료 즉시 반영 확인
```

#### [흐름 2] 점검 모드 활성화

```
점검 모드 토글 활성화 → [Save Changes] 클릭
  └─► 일반 사용자 서비스 접속 시 점검 안내 페이지 표시
        └─► 점검 모드 비활성화 후 저장 → 서비스 정상 접속 복구
```

## 10. 고객 문의

**경로:** `/admin/inquiries`  
**접근 권한:** 어드민 전용  
**목적:** 접수된 고객 문의를 조회하고 답변 처리한다

---

### 기능 목록

| #   | 기능          | 설명                                                  |
| --- | ----------- | --------------------------------------------------- |
| 1   | 사이드바 메뉴 접근  | 사이드바 메뉴를 통해 고객 문의 페이지(/admin/inquiries)에 접근할 수 있다   |
| 2   | 문의 목록 조회    | 접수된 문의 목록이 표시된다 (접수 상태, 작성자 등 정보 포함)                |
| 3   | 문의 상태별 분류   | 답변 대기 중인 문의와 보관함에 저장된 문의가 각각 별도 영역에서 조회된다           |
| 4   | 문의 상세 조회    | 개별 고객 문의의 상세 정보를 확인할 수 있다                           |
| 5   | 답변 완료 처리    | 답변 완료 처리 버튼 클릭 시 상태가 변경되고 처리 결과가 저장된다               |
| 6   | 보관함 이동 처리   | 보관함 이동 처리 버튼 클릭 시 해당 문의가 보관함으로 이동되고 결과가 저장된다        |
| 7   | 문의 응답 방식 안내 | 문의 응답을 위한 폼이 존재하거나, 별도 이메일 클라이언트로 전송해야 한다는 안내를 제공한다 |

---

### 사용자 흐름

#### [흐름 1] 문의 조회 및 답변 처리

```
사이드바 메뉴 → [고객 문의] 클릭 → /admin/inquiries 접속
  └─► 문의 목록에서 대기 중인 문의 확인
        └─► 개별 문의 클릭 → 상세 조회
              └─► [답변 완료 처리] 클릭 → 상태 변경 및 저장 완료 피드백
```

#### [흐름 2] 보관함 이동

```
문의 상세에서 [보관함 이동] 클릭
  └─► 보관함으로 이동 처리 완료
        └─► 보관함 영역에서 해당 문의 확인 가능
```
