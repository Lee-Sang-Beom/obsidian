## 0. 공통 / 어드민 접근 제어

> 어드민 페이지는 일반 사용자의 접근이 완전히 차단되어야 하며, 인증·인가 처리가 올바르게 되어 있어야 합니다.

### TC-A000 — 접근 제어 및 공통 UI

| #   | 검증 항목                                             | 결과  |
| --- | ------------------------------------------------- | :-: |
| 1   | 비로그인 상태에서 `/admin` 접근 시 로그인 페이지로 리다이렉트되는가         |  ✅  |
| 2   | 일반 사용자 계정과 접속이 완전히 분리되어 있는가                       |  ✅  |
| 3   | 제시된 계정으로 정상 로그인 및 어드민 콘솔 진입이 되는가                  |  ✅  |
| 4   | 로그아웃 버튼이 노출되며, 로그아웃 시, 로그인페이지로 이동하며 세션이 완전히 만료되는가 |  ✅  |
| 5   | 각 하위 메뉴(`/admin/*`) URL 직접 접근 시에도 권한 검증이 수행되는가    |  ✅  |
| 6   | 사이드바 네비게이션의 모든 메뉴 링크가 정상 동작하는가                    |  ✅  |
| 7   | 로그인 여부에 따라 상단 헤더의 적절한 컨텐츠 정보가 표시되는가               |  ❌  |
| 8   | 어두운 배경에 텍스트 색상 대비(Color Contrast)가 적절한가           | ⚠️  |
| 9   | 알림 발생 시, 정상적으로 헤더에 해당내용이 표시되는가                    | 🔲  |

- **판정:** ❌

- **오류내용:**
	- 로그인하지 않았음에도 헤더 영역에 'Super Admin' 접속 중, 로그아웃 버튼이 노출되어 있음
	- 로그인 중 로그인페이지로 접속 시 메인 페이지로 자동이동하지 않음.
	- 로고 클릭 시 메인 페이지 혹은 로그인 페이지로 이동할 수 있도록 개선 

- **확인필요:**
	- 알림 발생 시, 정상적으로 헤더에 해당내용이 표시되는지, 모두 읽음 버튼 클릭 시 읽음 처리가 진행되는지 확인 필요

| 6            | 방문자 추이 차트가 정상 렌더링 및 표시되는가              |  ✅  |

	- 긴급 거래 정지 버튼의 이벤트가 아예 발생하지 않음.
		- 만약, 이벤트 발생이 불가능한 상태라면 사용자가 상황을 인지할 수 있도록 별도 alert을 띄워주어야 함

- **확인필요:**
	- 거래액 및 트래픽 추이 영역: 실제 DB값 기반으로, 데이터가 있을 시 정상적으로 차트가 표시되는지 확인 필요
	- 에스크로 보관 자금에 대한 '자금 동결' 버튼에 대한 이벤트가 발생하지 않음. (기타 다른 항목은 작동 확인됨)
		- 현재 에스크로 보관 자금에 대한 자금 동결 클릭 시, 오류로 이벤트가 없는건지, 클릭할 수 없는건지 인지자체가 불가능.
		- 만약, 이벤트 발생이 불가능한 상태라면 사용자가 상황을 인지할 수 있도록 별도 alert을 띄워주어야 함

	- **서비스 페이지에서 어떤 동작을 해야 로그&보안 페이지에 데이터가 스택되는지 확인 필요**
	- CSV 다운로드 동작 자체는 정상적으로 작동함을 확인. 실 데이터 포함 시, 데이터가 누락없이 포함되어 다운로드되는지 추가 확인 필요

## 10. 고객 문의 (`/admin/inquiries`)

### TC-A010 — 문의/제안 관리

| #         | 검증 항목                                                    | 결과  |
| --------- | -------------------------------------------------------- | :-: |
| **기본 설정** |                                                          |     |
| 1         | 별도 메뉴로 해당 페이지가 접근 가능한가                                   |  ❌  |
| 2         | 접수 상태 및 작성자 등의 정보를 적절히 출력하며, 문의 목록이 출력되는가                |  ❌  |
| **상세 설정** |                                                          |     |
| 3         | 개별 고객문의 상세 정보를 해당 고객문의 페이지에서 확인할 수 있는가                   |  ✅  |
| 4         | 답변 완료 처리 버튼 클릭 시 적절한 처리 및 알림이 발생하는가                      |  ❌  |
| 5         | 보관함 이동 처리 버튼 클릭 시 적절한 처리 및 알림이 발생하는가                     |  ❌  |
| 6         | 문의 응답을 위한 Form이 존재하거나, 별도 이메일 클라이언트에서 전송해야한다는 안내를 진행했는가? |  ✅  |

- **판정:** ❌

- **오류내용:**
	- 대시보드 페이지가 아니면, 해당 페이지로 이동가능한 방법이 없음. (메뉴에 존재하지 않음)
	- 답변 완료 처리, 보관함 이동 처리 시 결과 알림 처리 없이 UI만 변경됨. 
		- 리스트의 Status 변경은 확인되나, 새로고침 시 다시 초기화됨
		- 네트워크 상에서는 404 오류 발생

- **확인필요:**
	- 서비스 페이지에서 문의 기능 개발이 완료되면, 상위 검증 항목 재검토 필요
	- 보관함에 저장된 문의내용과 보관되지않은 문의내용을 각각 다른 영역에서 볼 수 있도록 개선하는 방향 검토
		- 리스트가 많아질수록 답변해야 하는 문의 내용을 찾기 난잡해질 가능성이 있음.

## 📝 추가 테스트 고려 사항

- **실 데이터 확보 후 재검증 필요**: 거래 운영 관리(TC-A002), 분쟁 & 리스크(TC-A003), 로그 & 보안(TC-A008) 페이지는 실 데이터가 없어 대부분 항목이 미테스트 상태. 서비스 페이지에서 실제 거래·분쟁·이벤트를 발생시킨 후 재검증 필요
- **시스템 설정 저장 수정 후 연계 검증**: TC-A009 저장 기능 수정 완료 시, 수수료율 변경이 매출/정산 관리(TC-A004)에 즉시 반영되는지 함께 확인 필요
- **권한 실시간 검증**: TC-A006 이슈(어드민 권한 박탈 후 세션 유지 문제) 수정 시, 권한에 따른 관리자 기능이 정상적으로 블로킹되는지 확인 필요
- **고객 문의 서비스 연동**: 서비스 페이지 문의 기능 개발 완료 후 TC-A010 전체 항목 재검토 필요
- **알림 발송 전체 검증**: 반려·정지·정산 등 주요 이벤트에서 판매자/구매자에게 알림이 실제 발송되는지 이메일 수신 기준으로 통합 확인 필요
