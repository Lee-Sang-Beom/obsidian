## 1. 테스트 범위

  

**서비스**: H-Escrow (부동산 안전 거래 플랫폼)

**URL**: https://home-ascrow.vercel.app

  

| 구분 | 페이지 |

|------|--------|

| 공통 | 랜딩, 로그인, 회원가입 |


| 판매/임대 | 홈 대시보드, 매물 관리, 매물 등록(3단계), 거래 관리, 거래 상세 |

## 2. 버그 리포트 (Bug)

  

**[B-01] 거래 채팅방 — 구매인 접속요청 연속 전송 시 서버 에러**

- 위치: 거래 채팅방 > 구매인 접속요청 버튼

- 증상: 판매자가 구매자에게 접속요청을 연속으로 전송하면 Server Components 렌더 에러 발생 (`An error occurred in the Server Components render. [...] A digest property is included on this error instance.`)

  

**[B-02] 거래 채팅방 — 거래 취소 시 서버 에러**

- 위치: 거래 상세 > 거래 취소하기 버튼

- 증상: 구매자 결제 거부 후 판매자가 거래 취소하기 클릭 시 동일한 Server Components 에러 발생

  

**[B-03] 거래 채팅방 — 결제 재요청 버튼 무반응**

- 위치: 거래 채팅방 > 3단계 진행 중 구매자가 1회 결제를 거부한 상태에서, 판매자가 결제 재요청 시도 시

- 증상: '구매자가 결제를 거부했습니다. 사유: '사유내용'. 판매자가 조건을 재검토 후 새로운 결제 요청을 발송할 수 있습니다.' 메시지가 발생했음에도 판매자가 결제 재요청 버튼 클릭 시 아무 이벤트가 발생하지 않음

  

**[B-04] 매물 관리 — 임시저장 미작동**

- 위치: 매물 관리 > 매물 등록 > 임시 저장 버튼

- 증상: 임시저장 클릭 시점까지 입력한 내용이 정상 저장되지 않음

  

**[B-05] 공통 — 상단 알림 아이콘 클릭 시 이벤트 없음**

- 위치: 전체 페이지 상단 내비게이션 > 🔔 아이콘

- 증상: 클릭 시 아무 반응 없음 (개발 예정으로 확인됨)

## 3. 개선 제안 (Improvement)

  

**[I-01] 대시보드 — 오타 수정**

- `매물 탑색하기` → `매물 탐색하기`

  

**[I-02] 매물 등록 — 숫자 입력값 천 단위 콤마(,) 표시**

- 보증금, 월세 등 숫자 입력 필드에 자동 포맷팅 적용

  

**[I-03] 매물 등록 — 입력값 변환 표시 통일**

- 2단계(관리비 등)는 숫자 입력 시 아래에 `N만원` 등과 같은 가격 표시 있음

- 3단계(보증금·월세)는 없음 → 전 단계 통일 필요

  

**[I-04] 매물 관리 — 등록 중 매물 삭제 기능 추가**

- 완전히 작성되지 않은 `등록중` 상태의 매물을 목록에서 삭제할 수 있는 기능이 있으면 좋을 듯함

  

**[I-05] 매물 탐색 — 지역별 검색 필터 추가**

- 현재 유형·건물 필터만 존재. 지역(시/구/동) 기반 검색 필터 추가가 되면 좋을 듯함

  

**[I-06] AI 상담 — 대화 제목 수동 설정 기능 추가**

- 현재 대화 목록에 제목이 자동 생성되거나 `제목 없음`으로 표시됨 → 사용자가 직접 제목 편집 가능하도록 개선되면 좋을 듯함

  

**[I-07] 거래 채팅방 — 버튼과 콘텐츠 요소 간 시각적 구분 개선**

- 현재 클릭 가능한 버튼과 단순 정보 표시 박스의 시각적 차이가 불명확 → 어포던스 강화 필요 (ex. 호버 효과, 테두리 스타일 차별화)

- ex: 거래 채팅방 내에서 오른쪽 위의 AI문의, 구매인 접속요청, 거래요청 수락 글자 중 어떤 것이 버튼인지 한눈에 확인하기 힘듦

- ex: 거래 채팅방 내 오른쪽 중앙부분의 매물 상세보기, 계약서 정보가 버튼인지 한눈에 확인하기 힘듦
