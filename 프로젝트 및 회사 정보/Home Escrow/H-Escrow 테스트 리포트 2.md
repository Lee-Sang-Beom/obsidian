## 1. 버그 리포트 (Bug)
  
**[B-01] 임시저장 과정 발생 오류**
- 위치: (판매/임대 상태 전환 후) 매물 관리 > 매물 등록 
- 상황 및 증상
	- 매물 등록 누른 후, 임시저장 누르면 아래 에러 발생
	- 매물 등록 누른 후, 제목 입력 후 임시저장 누르면 아래 에러 발생
	- 1단계 기본 정보 입력 완료 후, '다음' 버튼을 누른 후 임시 저장 버튼을 누르면 아래 에러 발생
	- 현재 페이지가 아닌, 현재 위치한 단계에서 요구하는 **필수 입력요소**를 모두 입력하지 않고, 임시저장 버튼을 누르면 발생하는 것으로 예상
```plaintext
An error occurred in the Server Components render. The specific message is omitted in production builds to avoid leaking sensitive details. A digest property is included on this error instance which may provide additional details about the nature of the error.
```

**[B-02] 계약서 입력 정보 누락현상**
- 위치: (공통) 거래 채팅방
- 상황 및 증상
	- 임대인 특이사항 입력에서, 애완동물 키움여부(있음) ,거주자 수(1명), 결혼예정(예정) 입력을 진행한 상황
	- 위 입력사항이 진행되었음에도 계약서 미리보기에서 해당내용이 '특이사항 확인' 란에서 '답변 대기중'으로만 기록됨

**[B-03] 정산 단계 거래 취소 오류**
- 위치: (구매/임차 상태 전환 후) 내 거래 > 정산 요청 중인 거래 정보
- 상황 및 증상
	1. 매물 상세 페이지에서는 정상적으로 '정산이 진행 중이므로 취소할 수 없습니다'라고 안내되는 상황
	2. (문제발생) 구매자는 내 거래 페이지에서 '거래 채팅방 입장' 오른쪽의 더 보기 버튼을 눌러 거래 취소 버튼을 발견
	3. 해당 버튼을 누르면 네트워크 500에러가 발생하며 아무런 피드백 이벤트가 동작하지 않음

**[B-04] 매물 거래 취소 후에도 거래 채팅방 입장 가능한 문제**
- 위치: (공통) 양측 거래채팅방, 대시보드, 매물 탐색 
- 상황 및 증상
	1. 구매자가 등록된 매물에 거래 요청
	2. 판매자가 거래요청 수락
	3. 구매자는 대시보드에 '요청한 거래'에 대한 요청 상태와 거래방 입장이 가능한 상태
	4. 판매자가 등록된 매물의 상세 페이지로 가서 거래 취소버튼 클릭. (해당 시점부터 판매자는 거래채팅방 입장불가)
	5. (문제 발생) 구매자는 여전히 대시보드에서 요청한 거래 영역에서 거래방 입장이 가능하며, 대시보드에서는 해당 매물의 거래가 취소되었는지 알 수 없음


**[B-05] 구매자 입장에서 거래 중인 매물에 중복 거래요청을 할 수 있는 문제**
- 위치: (구매/임차 상태 전환 후) 매물 탐색 > 상세 매물 페이지
- 상황 및 증상
	1. 구매자가 등록된 매물에 거래 요청
	2. 판매자가 거래요청 수락
	3. 구매자는 대시보드에 '요청한 거래'에 대한 요청 상태와 거래방 입장이 가능한 상태
	4. (문제 발생) 이 때, 구매자가 매물 탐색 페이지로 이동해서 동일한 매물의 상세페이지로 이동하면 다시 거래 요청을 할 수 있음
	5. 동일 매물에 대한 거래 채팅방이 활성화되어있음에도 매물 상세페이지에서 중복된 거래요청 및 거래채팅방이 개설됨으로써 혼란을 불러일으킴


**[B-06] 거래 관리 페이지에 출력되는 시간이 올바른지 확인 필요**
- 위치: (구매/임차 상태 전환 후) 내 거래, (판매/임대 상태 전환 후) 거래 관리
- 상황 및 증상
	- 제목 왼쪽에 표시되는 시간이 올바른지 확인 필요 
	- 09:00~14:00 사이에 테스트를 진행했는데 00:00~04:00 범위로 표시됨
