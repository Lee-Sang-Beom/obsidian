## 1. 버그 리포트 (Bug)

**[B-01] 매물 등록 — 임시저장 과정 발생 오류**
![[Pasted image 20260320155105.png|356]]
- 위치: (판매/임대 전환 후) 매물 관리 > 매물 등록 > 임시저장 버튼
- 증상: 현재 단계의 필수 입력 요소를 모두 채우지 않은 상태에서 임시저장 클릭 시 Server Components 렌더 에러 발생. 아래 세 케이스에서 재현됨.
    - 매물 등록 진입 직후 임시저장 클릭 시
    - 제목만 입력 후 임시저장 클릭 시
    - 1단계 완료 후 '다음' 버튼 클릭 후 임시저장 클릭 시
    - 현재 단계에서 요구하는 필수 입력 요소를 모두 입력하지 않고 임시저장 버튼을 누르면 발생하는 것으로 예상

```plaintext
An error occurred in the Server Components render. The specific message is omitted in production builds to avoid leaking sensitive details. A digest property is included on this error instance which may provide additional details about the nature of the error.
```

**[B-02] 거래 채팅방 — 계약서 입력 정보 누락 현상**
![[Pasted image 20260320154405.png|364]]
![[Pasted image 20260320154425.png|369]]
- 위치: (공통) 거래 채팅방
- 증상: 임대인 특이사항(애완동물 있음, 거주자 수 1명, 결혼 예정)을 입력했음에도 계약서 미리보기의 '특이사항 확인' 란에 '답변 대기중'으로만 표시됨

**[B-03] 정산 단계 — 거래 취소 오류**
![[정산단계 거래 취소 오류.png|601]]
- 위치: (구매/임차 전환 후) 내 거래 > 정산 요청 중인 거래 정보
- 증상:
    1. 매물 상세 페이지에서는 '정산이 진행 중이므로 취소할 수 없습니다'라고 정상 안내됨
    2. (문제 발생) 내 거래 페이지에서 '거래 채팅방 입장' 오른쪽 더 보기 버튼을 통해 거래 취소 버튼 접근 가능
    3. 해당 버튼 클릭 시 네트워크 500 에러 발생, 아무런 피드백 없음

**[B-04] 거래 취소 후에도 구매자 측 거래 채팅방 입장 가능 문제**
![[거래 취소된 거래채팅방 입장 확인.png|946]]
- 위치: (공통) 양측 거래채팅방, 대시보드, 매물 탐색
- 증상:
    1. 구매자가 등록 매물에 거래 요청
    2. 판매자가 거래 요청 수락
    3. 구매자 대시보드에 '요청한 거래' 및 거래방 입장 버튼 표시됨
    4. 판매자가 매물 상세 페이지에서 거래 취소 클릭 (이후 판매자는 거래 채팅방 입장 불가)
    5. (문제 발생) 구매자는 여전히 대시보드에서 거래방 입장이 가능하며, 거래가 취소되었음을 대시보드에서 확인할 수 없음

**[B-05] 구매자 — 진행 중인 매물에 중복 거래 요청 가능 문제**
- 위치: (구매/임차 전환 후) 매물 탐색 > 매물 상세 페이지
- 증상:
    1. 구매자가 등록 매물에 거래 요청
    2. 판매자가 거래 요청 수락
    3. 구매자 대시보드에 '요청한 거래' 및 거래방 입장 버튼 표시됨
    4. (문제 발생) 구매자가 매물 탐색에서 동일 매물 상세 페이지로 이동 시 거래 요청 버튼이 재활성화되어 있음
    5. 중복 거래 요청 및 거래 채팅방이 추가 개설되어 혼란 유발

**[B-06] 거래 관리 — 표시 시간 정확성 검토**
![[Pasted image 20260320154655.png|405]]
- 위치: (구매/임차 전환 후) 내 거래, (판매/임대 전환 후) 거래 관리
- 증상: 거래 제목 왼쪽에 표시되는 시간이 실제 시간과 불일치. 09:00~14:00 사이에 테스트했으나 00:00~04:00 범위로 표시됨

## 2. 개선 제안 (Improvement)

**[I-00] 사이트 도움말 — 데모 영상 및 인터랙티브 체험 플로우 추가**

- 위치: (공통) 메인
- 내용: 스크린샷 외에 핵심 서비스 이용 흐름을 이해할 수 있는 데모 영상 또는 인터랙티브 체험 플로우 추가를 제안

**[I-01] AI 상담 — 대화 제목 수동 설정 기능 추가**

- 위치: (공통) AI 상담
- 내용: 과거 대화 목록에서 사용자가 직접 제목을 입력·수정할 수 있는 기능 추가

**[I-02] 마이페이지 — 노출된 내부 인증 방식(Clerk) 텍스트 표현 수정**
![[Pasted image 20260320154719.png]]
- 위치: (공통) 마이페이지 > 내 정보
- 내용: 내부 인증 provider 값(ex. Clerk 내부 식별자)이 그대로 노출됨. "이메일 회원가입", "구글 로그인" 등 사용자 친화적 표현으로 변환하여 표시 필요

**[I-03] 미인증 상태 — 거래 기능 접근 시 인증 유도 안내 추가**

- 위치: (공통) 구매/판매 동작을 포함하는 페이지 전체
- 내용: 휴대폰 또는 계좌 미인증 상태에서 거래 관련 기능 사용 시, 인증 안내 토스트를 노출하고 마이페이지 인증 화면으로 이동을 유도

**[I-04] 계좌 관리 — 구매/판매 모드와 분리하여 항상 접근 가능하도록 개선**

- 위치: 마이페이지 > 계좌 관리
- 내용: 현재 구매 모드에서는 계좌 관리가 마이페이지에 표시되지 않음. 모드와 무관하게 계좌 관리에 항상 접근 가능하도록 변경 제안

**[I-05] 거래 채팅방 — 알림 박스로 인한 채팅창 하단 가림 현상**
![[거래채팅방 - 채팅창 텍스트 잘림현상.png]]
- 위치: (공통) 거래 채팅방
- 내용: 상대방 미접속 시 표시되는 '상대방이 접속하지 않았습니다' 알림 박스가 채팅창 하단 내용을 가림. 상대방 입장 전에는 채팅방 자체가 의미 있는 상태가 아니어서 치명적이지는 않으나 UX 개선 필요

**[I-06] 거래 채팅방 — 우측 매물 요약 정보 텍스트 한글 통일**

- 위치: (공통) 거래 채팅방
- 내용: 방향 정보가 영문(ex. south)으로 표시됨. 매물 리스트 및 상세 페이지와 동일하게 한글(남향/북향 등)로 통일 제안

**[I-07] 거래 채팅방 — 과도하게 공백을 포함한 계약서 정보 버튼**
![[거래채팅방 - 계약서정보 과도한 공백.png]]
- 위치: (공통) 거래 채팅방
- 내용: 계약서 정보 조회를 위한 버튼의 좌측영역에 과도하게 들어간 공백 제거 제안

**[I-08] 알림 리스트 — 스크롤 및 삭제 기능 추가**
![[알림창 - 텍스트 끊김 문제.png]]
- 위치: (공통) 알림창 클릭 시 표시되는 알림 리스트
- 내용:
    - 알림이 많을 경우 하단 내용이 잘려 확인 불가. 스크롤 기능 추가 제안
    - 알림이 누적만 되는 구조. 개별 삭제 및 전체 삭제 기능 추가 제안

**[I-09] 알림 리스트 — 거래 요청 알림에 페이지 이동 버튼 추가**
![[거래 요청 알림에 대한 피드백 이동정보 추가 필요.png]]
- 위치: (판매/임대 전환 후) 알림창 > 알림 리스트
- 내용: 거래 요청 알림('[구매자 이름]님이 판매 매물 OO에 거래 요청을 보냈습니다') 수신 시 현재는 판매자가 직접 대시보드 또는 매물 상세 페이지로 이동해야 수락/거절 가능. 알림 항목에서 바로 해당 페이지로 이동할 수 있는 버튼 추가 제안
