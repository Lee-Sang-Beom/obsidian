## 1. 버그 리포트 (Bug)
  
**[B-01] 임시저장 과정 발생 오류**
- 위치: (판매/임대 상태 전환 후) 매물 관리 > 매물 등록 
- 증상
	- 매물 등록 누른 후, 임시저장 누르면 아래 에러 발생
	- 매물 등록 누른 후, 제목 입력 후 임시저장 누르면 아래 에러 발생
	- 1단계 기본 정보 입력 완료 후, '다음' 버튼을 누른 후 임시 저장 버튼을 누르면 아래 에러 발생
	- 현재 페이지가 아닌, 현재 위치한 단계에서 요구하는 **필수 입력요소**를 모두 입력하지 않고, 임시저장 버튼을 누르면 발생하는 것으로 예상
```plaintext
An error occurred in the Server Components render. The specific message is omitted in production builds to avoid leaking sensitive details. A digest property is included on this error instance which may provide additional details about the nature of the error.
```

## 2. 개선 제안 (Improvement)

****[I-00] 사이트 도움말**
- 위치: 메인
- 내용: 스크린샷 이외에도 특히 핵심 서비스 이용 시에 대한 이해도 향상을 위한 데모 영상이나 인터랙티브 체험 플로우를 추가. 

**[I-01] AI 상담 내용 제목 수정 관리**
- 위치: AI 상담
- 내용: 과거에 활성화했던 대화 내용에 제목을 별도로 기재할 수 있는 기능을 추가

**[I-02] 노출된 개발 내부 값(Clerk) 표현 수정**
- 위치: 마이페이지 > 내 정보
- 내용: 사용자에게 표시 시 "이메일 회원가입", "구글 로그인" 같은 사용자 친화적 표현으로 변경

**[I-03] 휴대폰 및 계좌 미인증 상태에서의 페이지 이용**
- 위치: 구매/판매 동작을 포함하는 페이지 전체
- 내용: 휴대폰 또는 계좌 미인증 상태에서 거래 관련 기능 사용 시, 인증 안내 토스트를 노출하고 마이페이지 인증 화면으로 이동을 유도

**[I-04] 구매/판매 모드별 계좌 등록 권한 분리 개선**
- 위치: 마이페이지 > 계좌 관리
- 내용: 현재 구매 모드 선택 시 계좌 관리 기능이 마이페이지에서 표시되지 않습니다. 계좌 관리를 구매/판매 모드와 분리하여 항상 접근 가능하도록 변경

**[I-05] 이미 등록된 계좌에 AUTH REQUIRED 표시**
- 위치: 마이페이지 > 계좌 관리
- 내용
	- 계좌 등록 시, 노란 경고 문구(AUTH REQUIRED)가 표시되어 사용자에게 혼란을 줌.
	- 해당 문구 표시를 변경할 방법이 현재 화면에서 제시되지 않음
	- 계좌 카드의 인증 상태를 정확히 반영하여 불필요한 경고 문구를 제거하거나, 미인증 항목을 명확히 안내하는 문구로 교체 고려

**[I-06] 거래채팅방 채팅창 하위 내용 텍스트 잘림 현상**
- 위치: 거래채팅방
- 내용
	- 상대방이 접속하지않았을 때 발생하는 '상대방이 접속하지 않았습니다' 알림박스가 채팅창의 하위부분을 가리는 문제
	- 그냥 내버려두어도, 채팅방 자체가 상대방이 입장한 후에 의미있어지는 요소이기때문에 크게 상관없을 듯 하지만, UX문제가 있다고 판단

**[I-07] 페이지 상단 알림창 출력**
- 위치: 로그인 이후 접근 가능한 모든 페이지
- 내용
	- 대시보드 페이지로 이동해야만 알림 아이콘을 확인할 수 있음
	- 홈화면에서도 로그인된 상태이면 알림 아이콘을 확인할 수 있도록 개선


**[I-08] 페이지 상단 알림창 클릭 시 발생하는 알림 리스트 영역**
- 위치: 알림창 클릭 시 발생하는 알림 리스트
- 내용
	- 알림 메시지가 많은 경우, 하위 내용이 잘려 확인할 수 없음
	- 아래쪽 내용도 확인할 수 있도록 스크롤 기능과 같은 대안 추가
