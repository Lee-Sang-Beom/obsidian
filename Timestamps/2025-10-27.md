- [x] 프로젝트 캐시버스터 작업
	- 시도 1: use-version-check.ts와 version.json을 이용한 방법으로 새로고침을 동반한 최신 리소스파일 로드 시도
		- 구현 방식: 앱 마운트 시 useEffect에서 version.json을 fetch하여 버전 비교 후 새로고침
		- 발생한 문제점들
			- SPA 라우팅 시 버전 체크 미작동: React Router의 `<Link>`컴포넌트는 클라이언트 사이드 라우팅만 수행하여 탭바 이동 시 version.json을 재요청하지 않음
			- 초기 로드 시에만 체크: 앱 마운트 시 한 번만 버전 확인하여 사용자가 앱을 켜놓은 상태에서 새 배포 발생 시 감지 불가
			- 백그라운드 복귀 시 미대응: 앱을 백그라운드로 보냈다가 포그라운드로 복귀할 때 버전 체크 로직 없음
			- 브라우저/웹뷰 캐싱 정책: version.json 자체가 브라우저에 캐시되어 cache-busting 쿼리 파라미터 없이는 구버전 정보 반환
			- Service Worker 간섭 가능성: PWA 환경에서 version.json조차 SW가 캐시하여 최신 버전 확인 실패
		- 결과: 탭바로 이동해도 새 버전의 UI가 로드되지 않고, 앱을 완전히 종료 후 재시작해야만 업데이트되는 문제 발생
