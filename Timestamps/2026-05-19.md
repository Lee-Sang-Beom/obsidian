#### Work (프로젝트: 딸깍)
---
- 백엔드/DB 관리 기술 스택 분석
	- **DB**: PostgreSQL
	- **ORM**: Drizzle
	    - 프로그래밍 언어의 객체와 RDB 테이블을 자동 매핑
	    - SQL과 유사한 문법으로 직관적
	    - TypeScript 타입 안정성이 뛰어난 경량 ORM


- 사용자 동작 및 반응 흐름 분석 (일반 제작 요청 예시)
```
[UI 컴포넌트] → [Server Action] → [Domain Service] → [Drizzle ORM] → [PostgreSQL]
```
 1. UI — `/dashboard/create`
	- 사용자가 폼 작성 후 생성 버튼 클릭
	- 연결된 Server Action 호출

 2. Server Action — `production-action.ts`
	- Clerk으로 유저 인증
	- Zod 스키마로 입력값 유효성 검증
	- 검증 통과 시 Domain Service 호출

 3. Domain Service — `production-service.ts`
	- UI/프레임워크에 종속되지 않는 순수 비즈니스 로직
	- DB 트랜잭션으로 복수 작업을 안전하게 묶어 Drizzle에 명령

 4. Drizzle ORM & DB — `src/lib/db/schema/...`
	- `db.insert()` 등의 명령을 SQL로 자동 변환
	- PostgreSQL에 실행하여 데이터 영구 저장

 5. 응답 및 화면 전환
	- 성공 시 생성된 `projectId` 등을 Server Action으로 반환
	- `redirect()` 로 다음 스텝 페이지(스크립트 작성 화면)로 이동

- 쿼리 키 관리 방식 분석
	- src/lib/query-keys.ts에서 종합관리
		- `lib` 디렉터리는 보통 **프레임워크/라이브러리 설정 및 유틸리티**를 담는데, 이건 순수 라이브러리 설정보다는 도메인 지식을 담고있음
		- `lib`에 두면 도메인 레이어가 인프라 설정 레이어에 의존하는 구조가 됨

	- 도메인 응집도는 지금과 비슷한 구조이되, 한눈에 파악 가능한 구조를 유지하기 위해서는 src/queries/query-keys.ts와 같이 세팅하거나, 규모 고려 시 도메인 별 분산을 진행해야 함
