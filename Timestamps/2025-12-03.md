- [x] 현재 tanstack query 사용에 대한 useSuspenseQuery 적용 가능성여부 검토
	- TanStack Query 현재 버전: `@tanstack/react-query": "^5.90.2"`
	- v5에서는 `useSuspenseQuery` 훅을 공식 지원하며, 기존 `useQuery` 대신 `import { useSuspenseQuery } from '@tanstack/react-query'`로 import하여 사용
	- `useSuspenseQuery`는 `isLoading`, `isError`를 반환하지 않으며, Suspense의 fallback과 ErrorBoundary가 해당 상태를 처리
	- **적용 불가 사유**: 현재 프로젝트의 `SimpleTable`과 `MobileTable` 컴포넌트는 `isLoading`과 `isError` props를 필수로 받아 내부에서 로딩 스켈레톤과 에러 UI를 직접 렌더링하는 구조로 구현되어 있음
		- 해당 테이블 컴포넌트들은 현재 다른 브랜치에서도 사용 중이며, `isLoading`/`isError`를 제거할 경우 Breaking Change가 발생하여 모든 사용처의 수정이 필요함
	- **결론**: 테이블 컴포넌트의 대규모 리팩토링 전까지는 기존 `useQuery` 방식을 유지하는 것이 안전하며, 향후 테이블 컴포넌트를 Suspense 호환 구조로 전환하는 시점에 `useSuspenseQuery` 도입을 재검토할 필요가 있음
