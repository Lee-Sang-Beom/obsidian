		- hooks/queries/chart/: 차트 데이터 조회 (시계열, 통계 등)
		- hooks/queries/signal/: 시그널 데이터 조회 (목록, 상세, 실시간 정보 등)
		- hooks/mutations/: useMutation을 사용한 POST/PUT/DELETE 요청 (특정 Service파일 함수 직접 호출)
		- hooks/business/: API 훅을 조합하여 비즈니스 로직 처리 
			- UI 표시데이터가 아닌 서버전송용 비즈니스 데이터를 만드는 로직 포함,
			- 조건부 데이터 변환
			- 서비스 훅 선택 및 특정 Service파일 함수를 호출하는 hook을 호출
		- hooks/ui/: 프론트엔드 내부에서만 사용하는 로직 (상태 관리, 이벤트 핸들러 등)
		- hooks/ui/modals/: 모달 관련 UI 로직
	- 변경 이유
		- 파일 수 증가로 인한 가독성 저하
		- 훅의 역할(API 호출 / 비즈니스 로직 / UI 로직)에 따른 명확한 분리 필요
		- React Query 패턴(queries/mutations)과 클린 아키텍처 원칙 적용
		- 도메인별 분류(chart/signal)로 데이터 성격에 따른 명확한 구분
		- 신규 훅 추가 시 배치 위치를 명확히 하여 일관성 확보
