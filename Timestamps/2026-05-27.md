	   - 모달 내 cropper 영역 고정 크기(600px) 또는 이미지 비율 기반 동적 height 적용


	- 크롭 과정에서 맞춤 기능 문제가 있는데 이걸 어떻게 했는가
		- 문제
		   - 가로/세로 맞춤 버튼 클릭 후 휠로 축소 시 이미지 상단이 잘려 복구 불가능한 상태가 됨
		   - 원인: viewMode=1의 "cropBox는 canvas 안에 있어야 한다" 규칙 때문
		     - 세로 긴 이미지를 가로 맞춤하면 canvas가 컨테이너 위아래로 삐져나옴
		     - cropBox 상단이 canvas 상단보다 위에 위치하게 되어 viewMode=1이 canvas를 강제로 위로 이동
		     - 이후 휠 축소 시 이미지 상단이 컨테이너 밖으로 밀려나 영구적으로 복구 불가
		       
		- 해결 시도 및 결론
		   - zoomTo() → setCanvasData() 직접 제어 방식으로 교체: 타이밍 이슈는 제거되나 근본 문제 미해결
		   - cropBox 클램핑 (Math.min) 적용: 맞춤 직후는 정상이나 이후 휠 조작 시 재발
		   - 동적 height 계산 (onReady): 이미지 비율 기반으로 cropper 높이를 자동 조정, viewMode=1 유지 가능
		     - 단, MAX_CROPPER_HEIGHT 초과하는 극단적으로 세로 긴 이미지는 여전히 잘릴 수 있음
		   - viewMode=0 + fillColor: '#ffffff'와 같은 해결방법을 사용할 수 있으나, viewMode가 1이어야 기본 요구사항에 맞출 수 있으므로 초기화버튼을 추가하여 초기상태로 복귀할 수 있도록 구성
